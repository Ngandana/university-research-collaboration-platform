# Agile User Stories

## Introduction

This document translates system requirements and use cases into Agile user stories for the University Research Collaboration Platform. These stories follow the INVEST principles and ensure traceability to previous assignments.

---

## User Stories Table

| Story ID | User Story                                                                                | Acceptance Criteria                                            | Priority |
| -------- | ----------------------------------------------------------------------------------------- | -------------------------------------------------------------- | -------- |
| US-001   | As a student, I want to log into the system so that I can access my research projects     | Valid credentials grant access; invalid credentials show error | High     |
| US-002   | As a supervisor, I want to create research projects so that I can manage student research | Project must include title and description                     | High     |
| US-003   | As a student, I want to join a project so that I can collaborate on research              | Only invited users can join                                    | High     |
| US-004   | As a researcher, I want to upload documents so that I can share research materials        | File must be validated and stored                              | High     |
| US-005   | As a supervisor, I want to assign tasks so that I can track student progress              | Task must include deadline and assignee                        | High     |
| US-006   | As a student, I want to update task status so that I can reflect my progress              | Status updates must be saved                                   | Medium   |
| US-007   | As a user, I want to send messages so that I can communicate with my team                 | Messages must be delivered instantly                           | Medium   |
| US-008   | As an admin, I want to manage users so that I can control system access                   | Admin can create/update/delete users                           | High     |
| US-009   | As management, I want to view reports so that I can monitor research performance          | Reports must display accurate data                             | Medium   |
| US-010   | As a system admin, I want data encrypted so that user information is secure               | All sensitive data must be encrypted                           | High     |
