# Template Analysis and Selection

## Introduction

This document evaluates GitHub project templates and selects the most suitable template for managing the University Research Collaboration Platform using Agile methodologies.

---

## Template Comparison Table

| Template         | Columns / Workflow                | Automation Features                       | Agile Suitability                         |
| ---------------- | --------------------------------- | ----------------------------------------- | ----------------------------------------- |
| Basic Kanban     | To Do, In Progress, Done          | No automation                             | Simple but limited for Agile              |
| Automated Kanban | To Do, In Progress, Done          | Auto-moves issues based on status changes | Highly suitable for Agile workflows       |
| Bug Triage       | New, Needs Review, Closed         | Focused on issue tracking                 | Suitable for maintenance, not development |
| Team Planning    | Backlog, Ready, In Progress, Done | Supports planning workflows               | Good for planning but less automation     |

---

## Selected Template: Automated Kanban

The **Automated Kanban template** was selected for this project.

### Justification

* Supports **automatic issue movement**, reducing manual work
* Aligns with **Agile principles** such as continuous delivery
* Works well with **GitHub Issues (user stories)**
* Enables **real-time tracking of sprint progress**

This template is ideal because the project already uses:

* User stories (Issues)
* Sprint planning (Milestones)
* Task tracking (Project Board)

Thus, Automated Kanban provides seamless integration and improves workflow efficiency.
