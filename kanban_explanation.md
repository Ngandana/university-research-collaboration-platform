# Kanban Board Explanation

## What is a Kanban Board?

A Kanban board is a visual project management tool used to track tasks as they move through different stages of a workflow. It uses columns such as "To Do", "In Progress", and "Done" to represent the status of tasks.

---

## How the Board Visualizes Workflow

The Kanban board created for this project includes the following columns:

* To Do: Tasks that have not yet started
* In Progress: Tasks currently being worked on
* Testing: Tasks under validation
* Blocked: Tasks that cannot proceed due to issues
* Done: Completed tasks

Each column represents a stage in the development lifecycle, allowing users to easily track progress.

---

## Work-In-Progress (WIP) Limits

To avoid bottlenecks, a WIP limit of **3 tasks** is applied to the "In Progress" column. This ensures that:

* Team members focus on fewer tasks at a time
* Tasks are completed faster
* Workflow remains efficient

---

## Support for Agile Principles

The Kanban board supports Agile methodologies by:

* Enabling continuous delivery of features
* Allowing flexible prioritization of tasks
* Providing real-time visibility into system development
* Encouraging iterative progress through task movement

Overall, the Kanban board improves transparency, efficiency, and collaboration within the development process.
